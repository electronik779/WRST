Если вы решите отказаться от режима Single-File (сжатия в один файл) и вернетесь к обычной публикации, где на выходе получается папка со 171 МБ файлов, разделение файлов на «обязательные» и «скачиваемые» подчиняется четкому правилу.
Все файлы из папки publish делятся на две категории.
------------------------------
## 1. Категория: Компоненты, которые можно удалить из папки и скачать при установке
Это системная среда выполнения .NET и официальные библиотеки Microsoft. Если при публикации вы отключите режим Self-contained (сделаете проект Framework-dependent), эти файлы вообще не появятся в вашей папке publish, что уменьшит её размер примерно на 80–100 МБ.
Вы можете полностью исключить их из раздачи, а в Inno Setup прописать их автоматическое скачивание. К ним относятся:

* Файлы ядра .NET: Все файлы, начинающиеся на System.*.dll, Microsoft.Extensions.*.dll, coreclr.dll, clrjit.dll.
* Файлы Windows App SDK / WinUI: Библиотеки графического движка Windows (например, Microsoft.InteractiveExperiences.Projection.dll, Microsoft.WinUI.dll).

Как это реализовать:
В команде сборки вы меняете флаг --self-contained true на --self-contained false. После этого папка «похудеет», а в скрипт Inno Setup (в секцию [Code]) вы добавляете строчку:

Dependency_AddDotNet10Desktop; // Скачает и установит .NET 10 Desktop Runtime, если его нет у пользователя

------------------------------
## 2. Категория: Файлы, которые ОБЯЗАТЕЛЬНО должны быть в установщике (скачать отдельно нельзя)
Эти файлы уникальны для вашего приложения. Их нет на серверах Microsoft, поэтому их невозможно «докачать» из интернета во время установки. Они должны физически находиться в папке с установленной программой:

* Главный исполняемый файл и его конфигурация: WRST.maui.exe и WRST.maui.runtimeconfig.json.
* Логика вашего приложения: Главная библиотека WRST.maui.dll, а также WRST.dll (проект WinForms, если MAUI-проект ссылается на него как на зависимость).
* Сторонние NuGet-пакеты: Если вы устанавливали сторонние библиотеки (например, Newtonsoft.Json.dll, Plugin.Maui.Audio.dll, библиотеки для работы с базами данных), они уникальны для вашего проекта и обязаны лежать рядом с .exe.
* Ресурсы (Assets): Папки с картинками, шрифтами, локализациями или встроенными базами данных (если они не зашиты внутрь ресурсов самого проекта).

------------------------------
## Чек-лист: Как навести порядок без сжатия
Если вы хотите поставлять приложение классической папкой, но без лишнего мусора, выполните перед упаковкой в Inno Setup следующие действия:

   1. Удалите файлы отладки (.pdb): Файл WRST.maui.pdb занимает много места и нужен только для поиска ошибок в коде. В установщик его включать нельзя.
   2. Удалите файлы XML-документации (.xml): Некоторые NuGet-пакеты копируют файлы справок (например, Newtonsoft.Json.xml). Они не влияют на работу программы, их можно смело удалить.
   3. Очистите папки локализации (языковые папки): Если в папке publish появилось много подпапок с именами de, es, fr, it, ja, zh-Hans и т.д., а ваше приложение переведено только на русский и английский, все остальные папки можно удалить вручную.
